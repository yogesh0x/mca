The development of the nine-class skin cancer detection system followed a structured multi-stage workflow
consisting of dataset preparation, class balancing, model training, backend deployment, and performance
verification. Each step was designed to create a reliable and reproducible pipeline for handling medical-
grade images and training a deep learning classifier.
3.1 Dataset Collection and Merging
Two different sources were used for assembling the dataset:
 HAM10000 dataset (used for AKIEC, BCC, DF, MEL, NV, VASC)
 Another dermatology image dataset (used for classes missing in HAM10000 such as SCC, PBK,
SK)
Since the number of samples per class varied significantly, you constructed a combined dataset with the

3.2 Dataset Balancing Through Augmentation
Because several classes had far fewer samples (some as low as 80), a dedicated augmentation script was
used to expand every class to exactly 500 images, ensuring uniform representation during training.
20
my augmentation script applied transformations such as rotation, shifting, shear,
zoom, and flips, and generated new images until the target count was reached.
The process was repeated automatically for every class folder.


The augmentation process ensured:
 Balanced dataset (500 images × 9 classes = 4,500 images total)
 Increased robustness of the model
 Reduced overfitting due to higher variety in training images
This balancing step was critical because deep learning models perform poorly when trained on imbalanced
datasets.
